Getting Started: Set Up Zotero Before You Start Reading

April 5, 2026 research tools

This is the first post here, so I wanted to start with something practical rather than an intro about myself. Specifically, something I wish I had set up earlier in my research: Zotero. If you're new to research or just starting a grad program, get it running before you start collecting papers.

What it is

Zotero is a free reference manager. It saves papers, organizes them, and handles citations. That description undersells it a bit, but that's the core of it. I came into my MS program trying to organize my references in a couple of different ways without creating one good system. Chaning the orgnazation for literature mid-project is not fun.

The Word plugin

Zotero has a plugin for Microsoft Word that lets you insert citations inline as you write. The bibliography builds itself and stays formatted for whatever style guide you're using. Switching between citation styles, say APA to IEEE, takes a few seconds. By the time you're writing a thesis chapter with 60 or more sources, managing citations by hand is just asking for errors. The plugin takes that whole problem off the table and saves a lot of time.

Organizing the literature

You can build collections inside Zotero organized by topic or project, so when you're working on a specific section of a literature review you can pull up just the relevant papers without scrolling through everything. You can also attach notes to individual papers, which is useful when you come back to something months later and need to remember why you saved it.

Tags

This is what I'd actually push people to think about when they start. Zotero lets you tag papers, and having a consistent set of tags from the beginning makes a real difference. Here's what I use:

  • skimmed - looked at the abstract and figures, haven't read it properly yet
  • completed - read all the way through
  • deeply reviewed - read carefully, took detailed notes, know it well
  • still need to review - flagged as important but haven't gotten to it
  • questions to answer - paper raised something I need to follow up on or look into further
  • interesting methods - the approach is worth coming back to for my own work
  • good background - useful for context and intro sections
  • not much here - read it, saved it, not contributing much to the project

The status tags (skimmed through still need to review) tell you where you are with a paper. The content tags (interesting methods, good background, questions to answer) tell you why it matters. Together they let you filter pretty specifically across multiple tags. If I need to find papers I've read closely that had methods worth borrowing, I can get that list in a few clicks.

The "not much here" tag is also more useful than it sounds. When you're deep in a review, you'll otherwise keep second-guessing papers you've already evaluated. Tag it, move on.

Start early

The library only works if it's complete. Starting it after you've already collected a bunch of papers means going back and adding everything manually, which is tedious. If you set it up at the beginning and save every paper into it from day one, the system stays useful without much effort to maintain it.

That's the first post. I'm planning to write about engineering topics, tools I use for research, and other things as I go. Thanks for reading.
